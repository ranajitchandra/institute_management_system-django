## Local setup (development)

1. Create a Python virtualenv and activate it:

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate
```

2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Configure environment (optional): copy sample env vars or set

- `SECRET_KEY` (for production)
- `DEBUG` (True/False)
- `DATABASE_URL` (optional; SQLite used by default)

4. Run migrations and collect static (for dev):

```bash
python manage.py migrate
python manage.py collectstatic --noinput
```

5. Create superuser (if you want a secure admin):

```bash
python manage.py createsuperuser
```

6. Run server:

```bash
python manage.py runserver
```

Visit http://127.0.0.1:8000 and log in.

## Deploying to Render (concise)

1. Add production deps to `requirements.txt`:
- gunicorn
- dj-database-url
- psycopg2-binary
- whitenoise
- python-dotenv (optional)

2. Update `IMSProject/settings.py` to use env vars:
- `SECRET_KEY = os.environ.get('SECRET_KEY')`
- `DEBUG = os.environ.get('DEBUG') == 'True'`
- `DATABASES = {'default': dj_database_url.config(conn_max_age=600, ssl_require=True)}`
- Add `whitenoise.middleware.WhiteNoiseMiddleware` to `MIDDLEWARE`
- Set `STATIC_ROOT` and `MEDIA_ROOT` and configure `STATIC_URL`/`MEDIA_URL`

3. Build/start commands for Render web service:

Build command:
```bash
pip install -r requirements.txt
python manage.py migrate --noinput
python manage.py collectstatic --noinput
```
Start command:
```bash
gunicorn IMSProject.wsgi:application --bind 0.0.0.0:$PORT
```

4. On Render dashboard:
- Create a PostgreSQL managed DB (recommended) and copy its `DATABASE_URL`
- Create a Web Service, connect GitHub repo, set build & start commands
- Set environment variables: `SECRET_KEY`, `DEBUG=False`, `DATABASE_URL`, `ALLOWED_HOSTS` (your Render URL)
- Deploy and watch logs

5. Media files:
- Use S3 (recommended) with `django-storages` or Render persistent disk.

## Notes & Troubleshooting

- If static files 404: ensure `collectstatic` ran and `WhiteNoise` configured.
- If DB errors: check `DATABASE_URL` and that migrations ran.
- Replace demo credentials and rotate `SECRET_KEY` before production.
